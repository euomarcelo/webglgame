Relatório

DIG DUG 2

1. Introdução

        Como trabalho final para disciplina de Fundamentos de Computação Gráfica, desenvolvemos um jogo baseado no clássico de Arcade/NES Dig Dug 2.

Nesse jogo, o personagem pode abrir rachaduras na terra, deixando seus perseguidores mais lentos ou abrir fendas na terra, causando com isso deslizamentos que eliminam seus inimigos. Abaixo, o repositório com código do jogo atualizado e o site do jogo.

Repositório GIT com o código fonte: https://github.com/euomarcelo/webglgame

Site com a demo do jogo: https://digdug2000.herokuapp.com/        

2. Ambiente de desenvolvimento e ferramentas utilizadas

  • Javascript
  • Google Chrome (versão 42.0.2311.152)
  • Jquery (versão 3.0.0)
  • Three.js (rev. 78)
  • IDE Webstorm (v. 2016.1.3)
  • Photoshop CS6 / Mspaint

Optamos por utilizar WebGL no desenvolvimento do jogo, o que veio a se mostrar um desafio grande, em vista da pequena quantidade de documentação e exemplos disponíveis. A linguagem de desenvolvimento do jogo escolhida foi o Javascript e como bibliotecas auxiliares utilizamos Jquery (para o registro e manipulação de eventos do jogo) e Three.js, que possui um papel semelhante ao que a GLUT tem para o desenvolvimento em OpenGL (possui diversas primitivas de desenho e incorpora algumas facilidades, como materiais).

As texturas utilizadas foram feitas à mão no Photoshop.

3. Comandos

  • Tecla W (ou seta para cima): andar para frente;
  • Tecla S (ou seta para baixo): andar para trás;
  • Tecla A (ou seta para esquerda): gira o jogador 90 graus no sentido antihorário;
  • Tecla D (ou seta para direita): gira o jogador 90 graus no sentido horário;
  • Barra de espaço : cria rachadura;
  • Tecla F : empurra o inimigo;
  • Tecla V : alterna entre os diferentes tipos de câmera;

4. Jogabilidade

O jogo desenvolvido pode ser jogado com visão de cima, câmera em primeira pessoa ou terceira pessoa.

        O objetivo do jogo é desmoronar território utilizando rachaduras, eliminando assim os inimigos. Sempre que uma sequência de rachaduras e buracos separara completamente uma região, dividindo o terreno em duas partes, a região de menor área desmorona.

Para criar uma rachadura, o jogador deve se posicionar sobre um dos buracos. Somente é possível criar rachaduras que liguem um buraco a: uma rachadura existente, outro buraco ou o oceano.

        

        O jogo termina quando o jogador elimina todos inimigos ou quando morre caindo no oceano ou tocado por um inimigo.

Conclusão

        Tivemos a oportunidade de aprender WebGL, além do OpenGL que vimos durante esse semestre; muito do que aprendemos sobre OpenGL foi imprecindível quando tivemos de enfrentar as dificultades ao longo do projeto, como a falta de documentação disponível.

        Inúmeros problemas aconteceram ao longo do desenvolvimento. A biblioteca de Three.js não possui documentação de muitas funções ou possui documentação defasada em muitos anos em alguns casos, apesar do impressionante esforço da comunidade. Houve problemas também com o navegador Firefox, que simplesmente não consegue rodar jogos com a velocidade ideal, o que motivou a troca para o Google Chrome como ambiente para rodar o jogo. Aconteceram também memory leaks - alguns de até 100MB, o que praticamente esgotava o limite de memória do navegador para WebGL - quando tentamos deletar elementos da cenas; deletar elementos é um problema em aberto no repositório da Three.js (não existe método destrutor); optamos por não deletar elementos durante o runtime para contornar o problema.

        Apesar das dificuldades, o projeto proporcionou um ambiente propício e adequado para exercitarmos o que aprendemos sobre meshes, materiais, iluminação, frames, câmeras e perspectivas.
